## 为什么

当前系统仅支持进京证状态查询和到期提醒推送，用户仍需手动打开 App 续办。六环外进京证不限办理次数且审核几乎即时通过，续办流程高度模式化（固定地址、固定目的），非常适合自动化。在到期前一天自动续办可消除空窗期，避免用户忘记续办导致违规。

## 变更内容

- **新增**：六环外进京证自动续办服务，在当前证件到期前一天自动提交续办申请
- **新增**：续办申请的完整 API 调用链（车辆校验 → 驾驶人校验 → 获取可选日期 → 提交申请）
- **新增**：`config.yaml` 中 `plates` 下新增 `auto_renew` 配置块，用户声明续办所需的固定信息（目的地、进京目的等）
- **新增**：续办触发时间随机化机制，在指定时间窗口内随机选择执行时刻
- **新增**：续办结果推送通知（成功/失败均通知用户）
- **修改**：`stateList` 响应解析需提取更多字段（`vId`, `hpzl`, `elzsfkb`, `cllx` 等）供续办使用

## 功能 (Capabilities)

### 新增功能
- `auto-renew`: 六环外进京证自动续办服务，包含续办判断逻辑、API 调用链、随机时间调度和结果通知
- `renew-config`: 续办相关的配置模型和校验，定义 `auto_renew` 配置块的结构与合法值

## 影响

- **配置**：`config.yaml` 新增 `plates[].auto_renew` 配置段，`config.yaml.example` 需同步更新
- **配置模型**：`jjz_alert/config/config_models.py` 新增续办配置的 Pydantic/dataclass 模型
- **配置校验**：`jjz_alert/config/validation.py` 新增续办配置项的校验规则
- **JJZ 服务**：`jjz_alert/service/jjz/` 下新增续办服务模块，复用现有 `http_post` 和 `jjz_parse` 基础设施
- **解析层**：`jjz_parse.py` 需从 `stateList` 响应中提取额外字段供续办使用
- **通知服务**：复用现有推送通道，新增续办结果的消息模板
- **定时调度**：主流程或定时任务中集成续办触发逻辑
